# Korean Humanizer Rules

한국어 텍스트에서 AI가 생성한 흔적을 찾아 자연스러운 문체로 다듬는 편집 가이드다. 영어권의 Wikipedia "Signs of AI writing" 가이드를 기반으로 하되, 한국어 문법과 문체에 맞게 재구성했다.

이 파일은 한국어 라우트의 작업 흐름과 출력 형식만 담는다. 상세 패턴과 예시는 별도 파일로 분리되어 있다. 작업 시 입력 텍스트의 성격에 맞는 reference만 읽으면 된다.

작업 전에 모든 reference를 다 읽을 필요는 없다. 입력 텍스트의 성격에 따라 관련 카테고리만 읽는다. 텍스트 유형이 명확하지 않거나 처음 사용할 때는 모든 카테고리를 훑어본다.

형식만의 정리는 선택 사항이다. 굵은 강조나 목록 앞 라벨은 사용자가 요청했거나 대상 문서의 스타일 가이드가 요구할 때만 고친다. 단, 이모지 장식, 챗봇 잔재, 홍보성 문구, 억지 3개 나열과 함께 나타나는 경우에는 해당 활성 패턴의 일부로 다룬다.


## 작업 순서

텍스트를 받으면 다음 순서로 진행한다.

1. 아래에 나열된 패턴을 기준으로 AI 흔적을 찾는다
2. 문제가 있는 부분을 자연스러운 표현으로 고친다
3. 원래 의미는 그대로 유지한다
4. **원문의 말투를 그대로 유지한다.** 원문이 "-합니다" 체이면 수정본도 "-합니다" 체로, 원문이 "-한다" 체이면 수정본도 "-한다" 체로 쓴다. 이 skill이 교정하는 것은 AI 특유의 패턴이지, 경어법이나 문체 등급이 아니다.
5. 글의 의도된 톤에 맞춘다. 보고서, 블로그, 기술 문서 등 맥락에 따라 허용되는 표현 범위가 달라진다
6. 초안을 작성한 뒤 스스로에게 묻는다: "아래 글에서 AI가 쓴 것 같은 부분이 남아 있는가?"
7. 남은 흔적을 짧게 정리하고, 한 번 더 고친다


## 문체와 개성

AI 패턴을 제거하는 것만으로는 부족하다. 개성 없이 깔끔하기만 한 글도 AI처럼 보인다.

### 개성이 없는 글의 특징

모든 문장의 길이와 구조가 비슷하고, 의견 없이 사실만 나열한다. 불확실성이나 복합적인 감정을 인정하지 않으며, 적절한 상황에서도 1인칭을 쓰지 않는다. 결과적으로 위키백과 문서나 보도자료처럼 읽힌다.

### 개성을 넣는 방법

먼저 의견을 넣는다. 사실을 나열하는 대신 반응한다. "이건 좋은 접근이지만 실제 적용에서는 의문이 남는다"가 장단점을 중립적으로 나열하는 것보다 사람답다.

리듬도 중요하다. 짧은 문장과 좀 더 길게 풀어가는 문장을 섞어 단조로운 호흡을 피한다.

복잡한 감정도 인정한다. "인상적이면서도 약간 불안한 구석이 있다"가 "인상적이다"보다 솔직하다.

1인칭을 쓸 수 있으면 쓴다. 1인칭이 비전문적인 것은 아니다. "계속 신경 쓰이는 부분이 있다"는 실제 사람의 사고를 보여준다.

구체적으로 쓴다. "우려된다"보다는 "새벽 3시에 아무도 안 보는 사이에 agent가 계속 코드를 생성하고 있다는 게 묘하게 불편하다"가 낫다.


## 출력 형식

다음 순서로 제공한다.

1. **초안**
2. **AI가 쓴 것 같은 부분이 남아 있는가?** 에 대한 짧은 답변
3. **최종 수정본**
4. **적용된 항목 표.** 어느 패턴 항목이 어떤 부분에 적용되었는지 표로 정리한다. 컬럼은 다음과 같다.
   - 항목 번호: 위 패턴 목록의 번호
   - 항목 이름: 해당 패턴의 제목
   - 수정 전: 원문에서 해당된 부분
   - 수정 후: 고친 결과
   - 적용 횟수가 많아 표가 너무 길어지면 상위 20개만 보여주고, 그 아래에 "이 외에 N건 더 있음"으로 표시한다.
5. **변경 사항 요약**: 표에서 다루지 못한 구조적 변경이나 추가 메모

표 예시:

| 항목 번호 | 항목 이름 | 수정 전 | 수정 후 |
|---|---|---|---|
| 7 | LLM 특유의 과장 어휘 | tuning이 조합 폭발한다 | tuning 변수가 서로 얽혀서 어디서부터 봐야 할지 모르게 된다 |
| 7-1 | 의인화/증상-원인 혼동 | robot이 갑자기 뛰어오르려 한다 | controller가 급격한 명령을 낸다 |


## 작업 절차 체크리스트

1. 입력 텍스트를 꼼꼼히 읽는다
2. 텍스트 유형에 따라 관련 reference 파일을 읽는다
3. 위의 패턴을 기준으로 AI 흔적을 모두 찾는다
4. 문제가 있는 부분을 고친다
5. 고친 텍스트가 다음 조건을 만족하는지 확인한다
   - 소리 내어 읽었을 때 자연스럽다
   - 문장 구조가 단조롭지 않다
   - 막연한 주장 대신 구체적인 사실이 들어 있다
   - 글의 맥락에 맞는 톤이다
   - 간결한 서술을 적절히 사용했다
6. 초안을 만든다
7. 스스로에게 묻는다: "아래 글에서 AI가 쓴 것 같은 부분이 남아 있는가?"
8. 남은 부분을 짧게 적는다
9. 최종 수정본과 적용된 항목 표를 만든다
