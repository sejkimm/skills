# System Archive Route

## Use This Route When

The user asks about Obsidian configuration, trash, archive, recovery, discarded notes, root files, or system-area routing.

## Do Not Use This Route When

- The user wants to create a normal note.
- The item clearly belongs to a learning, source, course, career, project, asset, or template route.

## System Areas

Treat application configuration and discarded files as operational areas, not knowledge taxonomy areas.

Rules:

- Do not create user notes in app-configuration areas.
- Do not use trash as a curated archive.
- Recover discarded material only when the user explicitly asks.
- After recovery, route the item through the normal router.
- If the user wants a real archive, use or propose a normal archive area rather than relying on trash.

## Outlier Handling

- Useful discarded learning material should be recovered and routed.
- Duplicate discarded source notes can remain discarded unless explicitly recovered.
- System/plugin files should not be moved into content folders.
